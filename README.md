# Chess AI

**Live demo:** https://chess-bot-deployment.vercel.app/

Full-stack chess AI: a **PyTorch** neural network trained on **Stockfish**-labeled positions for evaluation, paired with a classical alpha-beta search engine for move selection. **Flask** REST API on the backend, **React / TypeScript** frontend with a live evaluation bar and candidate-move arrows. Containerized with **Docker** (Render-deployable) and currently live on **Vercel** as a serverless Python function plus a static React build.

## Architecture

### PyTorch evaluator — `train_advanced_model.py`, `chess_model_improved.pth`

A fully connected neural network with shape **768 → 256 → 128 → 1** and ReLU activations. Each position is encoded as 12 piece-type planes (768 binary features) capturing where every piece sits for both colors, plus hand-crafted positional signals — material balance, mobility, castling rights, and center control.

Training data was generated by `python-chess` simulating thousands of legal board positions through random move sampling. Each position was labeled with the Stockfish chess engine's evaluation as the ground-truth target. The network was optimized with the **Adam** optimizer and **MSE** loss, with **dropout** and **batch normalization** for regularization, across multiple training iterations targeting improved validation loss.

### Classical search engine — `engine.py`

Iterative-deepening **negamax with alpha-beta pruning**, quiescence search on captures and queen promotions, MVV-LVA capture ordering with killer-move and history heuristics, a Zobrist-keyed transposition table, and a check extension. Evaluation is tapered between a midgame and endgame score using a phase weight derived from non-pawn material, blending material, piece-square tables, pawn structure (doubled, isolated), bishop pair, and a king-shield safety term.

Threefold-repetition and 50-move-rule draws are scored as 0 inside the search tree, so the engine actively refuses to repeat when winning and accepts the draw when losing.

### Opening repertoire

Eight curated openings — Italian Game, Ruy Lopez, Sicilian Defense, French Defense, Caro-Kann, Queen's Gambit, King's Indian Defense, English Opening — compiled at startup from canonical move lines covering both colors. The user picks one from a dropdown in the UI and the bot follows the line until either side deviates, at which point the search takes over.

## Stack

| Layer | Tech |
|---|---|
| Backend | Python, Flask, [`python-chess`](https://python-chess.readthedocs.io/) |
| Frontend | React, TypeScript, [`react-chessboard`](https://github.com/Clariity/react-chessboard), [`chess.js`](https://github.com/jhlywa/chess.js) |
| Training | PyTorch, NumPy, Stockfish (for ground-truth labels) |
| Deploy | Vercel (serverless Python function + static React build); Docker image for Render |

## Repo layout

```
engine.py                  # Move-selection engine (negamax + α-β + quiescence + TT + opening books)
backend_improved.py        # Flask REST API (used for the Docker / Render deploy path)
api/index.py               # Same Flask app, packaged as a Vercel serverless function
chess-frontend/            # React + TypeScript UI
train_advanced_model.py    # PyTorch training pipeline
chess_model_improved.pth   # Trained PyTorch weights
requirements.txt           # Runtime deps (flask, python-chess)
requirements-training.txt  # Training deps (torch, numpy, tqdm)
Dockerfile                 # Multi-stage build (React + Python backend)
vercel.json                # Vercel serverless function + static-build config
```

## API

| Method | Path | Body | Returns |
|---|---|---|---|
| `GET`  | `/api/health`           | — | service status |
| `GET`  | `/api/openings`         | — | list of named openings the bot can play |
| `POST` | `/api/evaluate`         | `{fen}` | white-POV evaluation in pawns, optional `mate_in` |
| `POST` | `/api/best-move`        | `{fen}` | best move (UCI + SAN) |
| `POST` | `/api/suggested-moves`  | `{fen, top_n?}` | top-N candidate moves with scores |
| `POST` | `/api/make-ai-move`     | `{fen, opening?}` | bot's move and resulting FEN |

## Local development

```bash
# Backend
python -m venv venv && source venv/bin/activate
pip install -r requirements.txt
python backend_improved.py            # serves on :5001

# Frontend (separate terminal)
cd chess-frontend
npm install
npm start                              # opens http://localhost:3000
```

To re-run the training pipeline, install the training deps and run the script (you'll need a local Stockfish binary on `PATH` for label generation):

```bash
pip install -r requirements-training.txt
python train_advanced_model.py
```
